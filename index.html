Insecure File Upload

Checklist singkat untuk memastikan fitur upload di aplikasi PHP tetap aman.

Kenapa ini penting?

Dampak dari upload berbahaya

  • Remote Code Execution dan pengambilalihan server.
  • Eksfiltrasi konfigurasi/databases sensitif.
  • Pivoting ke server lain di jaringan internal.
Langkah inti

Alur upload aman

1. Validasi input

Pastikan ada file, ukuran masuk batas, dan tidak ada error `$_FILES`.

2. Cek tipe asli

Gunakan `finfo_file()` & signature (magic bytes), bukan ekstensi dari user.

3. Sanitasi konten

Untuk gambar, gunakan `getimagesize()` agar skrip terselubung gagal.

4. Simpan aman

Letakkan file di luar web root, rename pakai ID random, log setiap upload.

Best practice tambahan
  • Batasi hanya tipe yang dibutuhkan (JPG/PNG) dan enforce `Content-Type` saat serving.
  • Gunakan sandbox (AppArmor/SELinux) serta nonaktifkan eksekusi di folder upload.
  • Monitoring ukuran & frekuensi upload untuk mendeteksi perilaku abnormal.
